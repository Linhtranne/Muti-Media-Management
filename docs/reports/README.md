# Task Reports

This directory contains post-work reports generated by AI agents after completing tasks.

## Folder Convention

Do not place new report files directly in `docs/reports`.

Use the most specific folder:

```text
docs/reports/US-XXX/REPORT-{story-or-task-slug}-{YYYY-MM-DD}.md
docs/reports/AI-SDLC-XXX/REPORT-{story-or-task-slug}.md
docs/reports/AI-SDLC/REPORT-{workflow-or-governance-slug}-{YYYY-MM-DD}.md
docs/reports/OPS/REPORT-{ops-or-maintenance-slug}-{YYYY-MM-DD}.md
```

Examples:

- `docs/reports/US-016/REPORT-US-016-plan-setup-2026-07-01.md`
- `docs/reports/US-017/REPORT-US-017-plan-setup-2026-07-01.md`
- `docs/reports/AI-SDLC/REPORT-docs-retrofit-2026-06-29.md`
- `docs/reports/OPS/REPORT-eslint-cleanup-2026-06-24.md`

## Naming Convention

```text
REPORT-{task-slug}-{YYYY-MM-DD}.md
```

Story reports should include the story id in the filename unless the parent folder already has a stricter convention.

## Report Template

Each report includes:

1. **Summary** - What was accomplished
2. **What Was Done** - Itemized list
3. **How It Was Done** - Approach, tools, skills
4. **Files Changed** - Path, action, description
5. **Impact & Purpose** - System-level significance
6. **Decisions Made** - Rationale and alternatives
7. **Verification** - Tests, docs, secrets, ACs
8. **Open Items** - Next steps
9. **AI-SDLC Completion Gate** - Required for story-level AI-SDLC artifacts
